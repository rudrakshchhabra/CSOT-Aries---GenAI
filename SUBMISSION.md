						CSOT GEN AI WEEK- 3


This week, I rebuilt our basic AI prototype into a fully functioning, modular Research Desk. Honestly, it was a massive jump from what we did before. The main focus was breaking down a giant, messy script into a clean architecture. I separated the core agent logic (the brain) from the actual tools (like web fetching, reading HuggingFace papers, and editing files) and put the full-screen terminal interface in its own file (tui.py). At first, moving all this code around just felt confusing, but I finally realized it’s all about "separation of concerns"—it makes finding and fixing bugs way easier when every file only has one specific job. I also got persistent memory working, so the agent saves every chat into JSON files and actually remembers previous turns. The most frustrating part of the project wasn't even the Python code itself; it was dealing with the AI API. I spent way too much time fighting with OpenRouter's free tier. I kept getting random 404 and 400 errors because their servers were constantly dropping or renaming the free models. But that actually ended up being a great stress-test for the app. Instead of the whole program crashing, the asynchronous threading in my Textual UI caught the API errors and printed them cleanly to the screen. I made the conscious choice to skip the bonus challenges this week (like auto-titling the sessions) because I wanted to take a step back and make sure I actually understood how the tool routing and the background threading worked under the hood before piling on extra features. By the end of it, getting this architecture running finally felt like I was putting together real software instead of just writing a simple script.
